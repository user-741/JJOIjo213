Network Topologies

Star Network

Central hub, fast but vulnerable at center

Bus Network

Simple, no central node, collision-prone

Ring Network

Closed loop, resilient to single node failure

These "classics" underpin modern mesh networking. IoT often revisits ring/bus concepts in constrained networks.

2. Hardware Components

NIC

Network Interface Card - provides unique MAC address

Hub

Dumb broadcast device (Layer 1)

Switch

Smart forwarding based on MAC (Layer 2)

Router

Inter-networking + routing decisions (Layer 3)

Switches are the workhorses of LAN; routers shape traffic between networks and enforce policies.

3. OSI Model in Wireless Context

Layer 7: Application

HTTP, FTP, SMTP - User-facing protocols

Layer 6: Presentation

Format translations, encryption

Layer 5: Session

Manages wireless disruptions, session continuity

Layer 4: Transport

TCP/UDP reliability

Layer 3: Network

IP routing between networks

Layer 2: Data Link

Access control, error handling

Layer 1: Physical

Waves/light - vulnerable to interference

Wireless-specific attacks target Layers 2-4 (jamming, spoofing, replay, session hijack). Privacy-preserving methods operate at Layers 5-7.

Session 2: Cyber Physical Systems

1. What are Cyber Physical Systems?

CPS integrates computation, networking, and physical processes. They represent the next computing revolution after mainframe, desktop, and ubiquitous computing.

Perception Layer

Sensors, RFID, WSN, Smart Cards

Data Collection & Environment Monitoring

Transmission Layer

Wireless/Wired Networks

Data Transport & Communication

Application Layer

Intelligent Processing

Decision Making & Control

CPS categories: Infrastructural CPSs (emphasize physical side, add cyber for control) vs Personal CPSs (emphasize cyber side, add physical for enhanced utility).

2. CPS Applications

Smart Grid

Real-time cooperative control, self-healing power systems

e-Transportation

V2V networks, autonomous navigation, 30-90 processors per car

Smart Healthcare

Electronic Patient Records, home monitoring, robotic surgery

Industrial Control

SCADA systems, process automation, critical infrastructure

3. CPS Security Fundamentals

CIAA Triad Extended

Confidentiality

Only authorized parties access assets

Integrity

Assets modified only by authorized parties

Availability

Assets accessible to authorized parties

Authenticity

Data, transactions are genuine

CPS security cannot simply use conventional cyber security schemes - solutions must integrate with underlying physical process control features.

4. CPS Attacks & Threats

Physical Layer Attacks

  • Node Capture: Control over nodes, steal encryption keys
  • False Node: Addition of malicious nodes sending false data
  • Node Outage: Stopping node services, causing DoS

Network Layer Attacks

  • Wormhole: False routing paths creation
  • Jamming: Intentional interference causing DoS
  • Sinkhole: Malicious routing announcements
  • Selective Forwarding: Selective packet dropping

Application Layer Attacks

  • Buffer Overflow: Exploiting software vulnerabilities
  • Malicious Code: Viruses, worms causing network slowdown

5. Defense Mechanisms

Prevention Strategies

  • Authentication and access controls
  • Security policies and network segmentation
  • Cryptographic approaches and encryption

Detection Strategies

  • Signature-based and behavior-based IDS
  • Anomaly detection algorithms
  • Real-time monitoring systems

Response Mechanisms

  • Automatic mitigation without human intervention
  • Self-healing and adaptive systems
  • Real-time constraint handling

Q: Which OSI layer handles routing between networks?
A: Network layer (Layer 3). It provides routing of packets through networks from source to destination using protocols like IP.
Q: Why are wireless LANs inherently less secure than wired LANs from a fundamental perspective?
A: Because wireless signals travel through air and can be intercepted by anyone within radio range, unlike wired signals confined to physical cables.
Q: What device intelligently forwards data frames based on MAC addresses?
A: A switch. It operates at Layer 2 and maintains a MAC address table to forward frames only to the intended recipient.
Q: What are the three main CPS architecture layers and their primary functions?
A: 1) Perception Layer (sensors, data collection), 2) Transmission Layer (networking, data transport), 3) Application Layer (intelligent processing, decision making)
Q: What is the difference between infrastructural and personal CPSs in terms of primary concerns?
A: Infrastructural CPSs: Availability is primary concern (critical infrastructure like power/water). Personal CPSs: Privacy is primary concern (personal data collection and tracking).
Q: What are the four components of the extended CIAA security triad for CPS?
A: Confidentiality (authorized access only), Integrity (authorized modifications only), Availability (accessible when needed), Authenticity (genuine data/transactions).
Q: Why can't conventional cyber security schemes be directly applied to CPS protection?
A: Because CPS security solutions must be closely integrated with underlying physical process control features. The cyber-physical interaction requires specialized approaches.
Q: What is a wormhole attack in CPS networks?
A: False paths are announced through which all packets are routed, creating information holes in the network and potentially allowing traffic interception.
Q: What is the primary challenge in wireless middleware for IoT systems?
A: Balancing efficiency with security while ensuring privacy guarantees - optimizations often conflict with privacy-preserving mechanisms.
